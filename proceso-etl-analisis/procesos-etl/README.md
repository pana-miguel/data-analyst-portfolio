
## 🧩 ¿Qué es un proceso ETL?

ETL significa **Extract, Transform, Load (Extraer, Transformar y
Cargar)**. Es un proceso fundamental en ingeniería de datos que permite
preparar la información para su análisis.

### 🔹 1. Extracción (Extract)

Consiste en obtener datos desde diferentes fuentes: - Bases de datos
(MySQL, PostgreSQL) - Archivos (CSV, Excel, JSON) - APIs - Sistemas
externos

### 🔹 2. Transformación (Transform)

En esta etapa los datos se limpian y se preparan: - Eliminación de
valores nulos - Corrección de errores - Normalización de datos -
Creación de nuevas variables (feature engineering) - Filtrado y
agregaciones

### 🔹 3. Carga (Load)

Los datos transformados se almacenan en un destino final: - Data
Warehouse - Base de datos - Herramientas de visualización (BI)

------------------------------------------------------------------------

## ⚙️ ¿Para qué sirve un proceso ETL?

-   Integrar datos de múltiples fuentes
-   Mejorar la calidad de la información
-   Facilitar el análisis de datos
-   Automatizar flujos de trabajo
-   Preparar datos para Machine Learning o BI

------------------------------------------------------------------------

## 🛠️ Herramientas comunes

-   Python (Pandas, Polars)
-   SQL
-   Power BI / Looker Studio
-   Excel

------------------------------------------------------------------------

## 💡 ¿Cómo ayuda el ETL al análisis?

Un buen proceso ETL: - Garantiza datos limpios y confiables - Reduce
errores en el análisis - Acelera la toma de decisiones - Permite crear
dashboards más precisos
